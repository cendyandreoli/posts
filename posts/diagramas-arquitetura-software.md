# Visualizando Arquitetura de Software com Diagramas

Aprenda a criar diagramas interativos para documentar e comunicar a arquitetura do seu software de forma clara e efetiva.

---

Diagramas são essenciais para comunicar e documentar a arquitetura de software de forma clara.

## Por que usar diagramas?

Diagramas ajudam a:
- Comunicar ideias complexas de forma visual
- Documentar decisões de arquitetura
- Onboarding de novos membros
- Identificar problemas de design

## Fluxograma de Decisão

```mermaid
flowchart TD
    A[Início] --> B{Usuário autenticado?}
    B -->|Sim| C[Dashboard]
    B -->|Não| D[Login]
    D --> E{Credenciais válidas?}
    E -->|Sim| F[Criar sessão]
    E -->|Não| G[Mostrar erro]
    G --> D
    F --> C
    C --> H[Fim]
    
    style A fill:#4ade80
    style H fill:#f87171
    style C fill:#60a5fa
```

## Diagrama de Sequência

Mostra a interação entre componentes ao longo do tempo:

```mermaid
sequenceDiagram
    actor User
    participant Frontend1
    participant API
    participant Database
    participant Cache
    
    User->>Frontend: Clica em "Buscar"
    Frontend->>API: GET /posts?q=react
    API->>Cache: Verifica cache
    
    alt Cache hit
        Cache-->>API: Dados do cache
    else Cache miss
        API->>Database: SELECT * FROM posts
        Database-->>API: Resultados
        API->>Cache: Armazena no cache
    end
    
    API-->>Frontend: JSON response
    Frontend-->>User: Exibe resultados
```

## Diagrama de Classes

Representa a estrutura de classes do sistema:

```mermaid
classDiagram
    class User {
        +String id
        +String name
        +String email
        +updateProfile()
    }
    
    class Post {
        +String id
        +String title
        +String content
        +User author
        +publish()
    }
    
    class Comment {
        +String id
        +String content
        +User author
        +Post post
    }
    
    User "1" --> "*" Post : writes
    User "1" --> "*" Comment : writes
    Post "1" --> "*" Comment : has
```

## Diagrama de Estados

Útil para modelar ciclo de vida de entidades:

```mermaid
stateDiagram-v2
    [*] --> Draft
    Draft --> Review : submit
    Review --> Draft : reject
    Review --> Published : approve
    Published --> Archived : archive
    Archived --> Published : restore
    Archived --> [*] : delete
```

## Diagrama de Arquitetura

Visão geral de um sistema distribuído:

```mermaid
graph TB
    subgraph Frontend
        A[React App]
        B[Mobile App]
    end
    
    subgraph Backend
        C[API Gateway]
        D[Auth Service]
        E[Posts Service]
        F[Comments Service]
    end
    
    subgraph Data
        G[(PostgreSQL)]
        H[(Redis Cache)]
        I[S3 Storage]
    end
    
    A --> C
    B --> C
    C --> D
    C --> E
    C --> F
    E --> G
    E --> H
    F --> G
    D --> G
    E --> I
    
    style A fill:#61dafb
    style B fill:#61dafb
    style C fill:#ff6b6b
    style G fill:#336791
    style H fill:#dc382d
```

## Conclusão

Use diagramas para documentar seu software. Eles facilitam a comunicação e ajudam a manter o conhecimento da arquitetura acessível a toda a equipe.
